# AGI-Sales Driver 要件定義書

> 営業知能基盤『AGI-Sales Driver』構築指示書

## 1. コンセプト：Manager Freedom（マネージャーの解放）

知識教育・スキル添削・データ分析をすべてAIが代行し、マネージャーが「メンバーの心のケア、モチベーション管理、理念浸透」に100%集中できる環境を作る。

### 設計原則

| 原則 | 説明 |
|------|------|
| AI代行領域 | 知識教育、スキル添削、データ分析 |
| 人間集中領域 | 心のケア、モチベーション管理、理念浸透 |
| データ棲み分け | 販路別 × 商材別 × 会社別（マルチテナント） |

---

## 2. インプット層（AIに学習させる「餌」）

すべてのデータは「販路・商材別」かつ「会社別（マルチテナント）」に完全に棲み分けする。

### 2.1 社内知財

| データ種別 | 形式 | 説明 |
|-----------|------|------|
| 営業資料 | PDF, PPTX, DOCX | 会社が保有する提案書・パンフレット等 |
| トークスクリプト | テキスト | 最新の営業トークスクリプト |
| 商材マニュアル | PDF, テキスト | 各商材の詳細情報・FAQ |

### 2.2 現場生データ

| データ種別 | 形式 | 説明 |
|-----------|------|------|
| 商談録音データ | 音声ファイル（mp3, wav, m4a） | 商談の音声録音 |
| 商談議事録 | テキスト | 録音からの文字起こし or 手動入力 |
| 日報 | テキスト | 営業担当者の日次活動報告 |

### 2.3 勝敗エビデンス

| データ種別 | 形式 | 説明 |
|-----------|------|------|
| 受注要因（KSF） | 構造化データ | Key Success Factors の分析結果 |
| 失注要因（KFF） | 構造化データ | Key Failure Factors の分析結果 |

### 2.4 データ棲み分け構造

```
テナント（会社）
  └── 販路（チャネル）
       ├── テレアポ
       ├── 飛び込み
       ├── Web反響
       ├── 紹介
       ├── セミナー
       └── 展示会
            └── 商材
                 ├── 商材A
                 ├── 商材B
                 └── 商材C
                      └── データ
                           ├── 社内知財
                           ├── 現場生データ
                           └── 勝敗エビデンス
```

## 3. アウトプット層（営業マンへの「武器」）

### 3.1 Quick FAQ

| 項目 | 内容 |
|------|------|
| 目的 | 資料を読み解く手間を省き、即座に「正しい答え」を提示 |
| 入力 | 営業担当者の質問（テキスト） |
| 出力 | インプット層の社内知財・マニュアルに基づく回答 |
| フィルタ | 販路別・商材別に回答を最適化 |
| 免責 | 回答末尾に「最新データは担当者に確認」の定型文 |

### 3.2 Smart Script（旧: Closing-AI）

| 項目 | 内容 |
|------|------|
| 目的 | 録音分析に基づき「今、その商材で最も刺さる」最新スクリプトを自動生成 |
| 入力 | 商談録音分析結果、KSF/KFF、販路・商材選択 |
| 出力 | 最新トークスクリプト、反論処理パターン、トレンドフレーズ |
| 更新頻度 | 録音データ蓄積に応じてリアルタイム更新 |
| 差別化 | 成功事例の話法を自動抽出し反映 |

### 3.3 Interactive Roleplay

| 項目 | 内容 |
|------|------|
| 目的 | 「失注要因」を学習したAI顧客を相手に24時間特訓 |
| 形式 | 音声対話型（将来）、テキスト対話型（初期） |
| AI顧客タイプ | 価格重視型、品質重視型、比較検討型、即決型、慎重型 |
| 難易度 | 初級、中級、上級 |
| フィードバック | ターンごとのスコアリング、弱点分析 |
| 学習ソース | KFFデータから失注パターンを再現 |

### 3.4 Auto-Sales Doc（営業資料自動生成）

| 項目 | 内容 |
|------|------|
| 目的 | 商談状況に合わせ、顧客にそのまま送れる「パーソナライズ提案資料」を自動生成 |
| 入力 | 商談情報（顧客名、業種、課題、予算感等） |
| 出力 | PDF形式の提案資料 |
| テンプレート | 会社別・商材別のベーステンプレート |
| パーソナライズ要素 | 顧客の課題、業種特有の事例、価格プラン |

## 4. 成長支援ツール

### 4.1 Sales-DNA（営業タイプ診断）

| 項目 | 内容 |
|------|------|
| 目的 | 営業担当者の性格タイプに最適化されたスクリプトを提供 |
| 診断方式 | 16タイプ診断（論理/共感等の軸） |
| 出力 | 性格タイプに合わせたスクリプト自動変換 |
| 適用範囲 | Smart Scriptで生成される全スクリプトに適用 |
| 例 | 論理型 → データ重視の話法、共感型 → ストーリー重視の話法 |

### 4.2 Daily-Career（ガクチカ・自己分析）

| 項目 | 内容 |
|------|------|
| 目的 | 日報から「ビジネススキル」を抽出し、就活用の成果ポートフォリオを自動蓄積 |
| 入力 | 日報データ |
| 出力 | スキル抽出、ポートフォリオ項目、市場価値スコア |
| 対象 | 学生インターン・新人営業 |
| スキルカテゴリ | コミュニケーション、提案力、分析力、粘り強さ、リーダーシップ等 |

## 5. 管理画面

### 5.1 AI Issue Detective

| 項目 | 内容 |
|------|------|
| 目的 | メンバーの「技術的な課題」や「失注傾向」をAIが自動分析しレポート化 |
| 入力 | メンバーの全活動データ（日報、商談録音分析、KSF/KFF） |
| 出力 | 課題レポート（メンバー別、チーム別） |
| 分析項目 | 失注傾向パターン、スキルギャップ、改善提案 |
| 更新頻度 | 日次自動分析 |

### 5.2 Human Touch Focus

| 項目 | 内容 |
|------|------|
| 目的 | AIが分析できない「心の折れ」「熱量の変化」の兆候を管理者にアラート |
| 検知手法 | 活動量の推移（日報頻度、商談数、ログイン頻度等） |
| 出力 | 「誰と面談すべきか」の可視化、アラート通知 |
| アラート条件 | 活動量の急減、日報未提出、成約率の急落等 |
| 表示 | ダッシュボードにアラートカード表示 |

### 5.3 管理者共通要件

| 項目 | 内容 |
|------|------|
| 全データ閲覧 | 各営業マンのインプット・アウトプットすべて閲覧可能 |
| Google Drive連携 | 管理者のGoogle Driveに情報が自動蓄積される |
| レポートエクスポート | PDF/CSV形式でのエクスポート |

---

## 6. 技術的ガードレール

### 6.1 免責事項

| 項目 | 内容 |
|------|------|
| 表示位置 | 全AI回答の末尾 |
| 定型文 | 「※本回答はAIによる分析結果です。最新のデータや詳細は担当者にご確認ください。」 |
| 適用範囲 | Quick FAQ、Smart Script、AI Issue Detective等すべてのAI出力 |

### 6.2 情報流出防止

| 対策 | 内容 |
|------|------|
| 権限分離 | テナント間の完全なデータ分離、ロールベースアクセス制御 |
| ID透かし | エクスポートされるドキュメントにユーザーIDの電子透かし |
| 大量コピー検知 | 異常な量のデータコピー・ダウンロードを検知しアラート |
| 監査ログ | 全操作の監査ログ記録 |

---

## 7. マルチテナント要件

| 項目 | 内容 |
|------|------|
| テナント単位 | 会社（代理店）単位 |
| データ分離 | テナント間でデータは完全に分離 |
| カスタマイズ | テナントごとに販路・商材を自由に設定可能 |
| ユーザーロール | admin（管理者）、field（営業担当者） |
| テナント管理 | 管理者がメンバーの追加・削除・権限変更可能 |

---

## 8. 日報機能（Daily Report）

### 8.1 営業担当者向け

| 項目 | 内容 |
|------|------|
| 入力項目 | 日付、顧客名、商談結果（成約/検討中/失注/初回訪問）、チャネル別KPI |
| AIコーチング | 日報送信後、AIが当日の活動を分析しフィードバック |
| ナレッジ抽出 | 成功パターンの自動抽出・ライブラリ蓄積 |

### 8.2 管理者向け

| 項目 | 内容 |
|------|------|
| 一覧表示 | メンバー全員の日報を時系列で閲覧 |
| 分析ダッシュボード | チーム全体のKPI推移、個人別パフォーマンス |
| Google Drive同期 | 日報データをGoogle Driveに自動保存 |

---

## 9. Google Drive連携

| 項目 | 内容 |
|------|------|
| 連携対象 | 管理者アカウントのGoogle Drive |
| 保存データ | 日報、商談録音文字起こし、分析レポート、生成スクリプト |
| フォルダ構造 | `AGI-Sales Driver/{テナント名}/{販路}/{商材}/` |
| 同期方式 | 自動同期（データ作成・更新時） |
| 認証 | Google OAuth 2.0 |
